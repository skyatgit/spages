# 📝 文档管理规范

## 规则

### ✅ 项目根目录
**只允许放置：**
- `README.md` - 项目主文档

### 📁 docs/ 目录（被 Git 忽略）
**所有其他文档都必须放在这里：**
- 技术设计文档
- 实现说明文档
- 测试指南
- 对比分析文档
- 开发笔记
- 架构说明
- API 文档
- 等等...

## 目录结构

```
spages/
├── README.md          ✅ 唯一允许在根目录的文档
├── .gitignore         （包含 docs/）
├── docs/              🚫 被 Git 忽略，不提交到仓库
│   ├── README.md      （docs 目录索引）
│   ├── *.md           （所有其他技术文档）
│   └── ...
├── src/
├── server/
└── ...
```

## 原因

1. **保持根目录整洁** - 避免文档文件过多造成混乱
2. **统一管理** - 所有文档集中在一个位置
3. **Git 忽略** - 开发文档不需要提交到代码仓库
4. **方便查找** - 知道所有文档都在 docs/ 目录下

## 执行规则

- ✅ README.md 在根目录
- ✅ 所有其他 .md 文档在 docs/ 目录
- ✅ docs/ 目录被 .gitignore 忽略
- ✅ 定期清理根目录的其他文档文件
